Internal Logic & Code Execution A. Map Generation (map/VoronoiGen.cpp & Climate.cpp) How to code it: Generation:  Scatter 20,000 points. Use jc_voronoi to build polygons. Tectonics:  Pick 10 random cells as plate centers. Expand outward. Where Plate A meets Plate B, push elevation to 1.0 (Mountains). Where they pull apart, drop to 0.0 (Ocean). Climate:  * Set Equator (Y = Center) temp to your user-defined max_temp. Set Poles (Y = 0, Y = Max) to min_temp. Cast wind vectors across the map. If wind hits a mountain cell, add moisture to the windward side and subtract moisture on the leeward side (rain shadow). Biome Assignment:  Loop through all cells. Match the cell's final temp/moisture against the BiomeDef structs to assign the biome tag (e.g., "TUNDRA"). B. The Auto-Populator (simulation/AutoPopulate.cpp) How to code it: Instead of running a 1,000-year sim, the user clicks "Auto-Populate". Settlements:  Find coastal or river cells with elevation < 0.4 and temperature > 0. Spawn a city. Assign it to a random Faction. * Roads (A  Pathfinding):** Run A* between cities. The "Cost" of moving from Cell A to Cell B is 1.0 + elevation_difference * 10. This forces roads through valleys and plains. Defenses:  If Cell A is owned by Faction 1, and neighbor Cell B is Faction 2, mark the edge between them as "Border_Wall". C. Import from Image (io/ImageImport.cpp) How to code it: User selects a grayscale .png. Use stbi_load to read the pixel data. Overlay the Voronoi grid on the image. For each cell, read the pixel brightness at its X/Y coordinate. elevation = pixel_brightness / 255.0. (White = Mountain, Black = Ocean). Proceed to Climate generation. 5. The Editor UI Layout (ImGui) The app is a fullscreen ImGui window split into three main areas. Left Panel (World Rules): Calendar Tab:  Input fields for "Era Name", "Year". ImGui::InputFloat for Moon cycles. ImGui::Checkbox for "Force Full Cycle". Climate Tab:  Sliders for Min/Max Temp at poles/equator, Sea Level threshold. Right Panel (Lore & Entities): On boot, sends HTTP GET to  Module 1 (Lore Database)  to fetch all known plants, animals, and cultures. Displays them in collapsible headers. You can click "Add to Map" or drag-and-drop a Faction onto a specific Voronoi cell. Center Canvas (The Viewport): Uses ImGui::GetWindowDrawList()->AddConvexPolyFilled() to physically draw the colored Voronoi cells. Contains brush tools (Increase Elevation, Paint Biome, Place Settlement). Bottom Panel (Simulation Log): When you hit "Simulate 500 Years", a std::vector<std::string> fills with historical events:  "Year 42: Ice Nomads attacked Northern Horde (Loved resource dispute: Iron)" . 6. The Final Output (The Export Contract) When the user is done, they click "Export Game Data". It writes Saga_Master_World.json using nlohmann. Notice how it explicitly contains the modular data you requested, ready for the Python engine.
