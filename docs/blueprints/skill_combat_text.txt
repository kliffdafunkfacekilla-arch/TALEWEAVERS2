Here are the developer-ready blueprints for  Batch 3 . 🏭 S.A.G.A. BLUEPRINT 06: The Skill Check Manager (The Fate Engine) 1. System Overview Definition:  A stateless Python microservice. It is responsible for resolving all non-combat friction. It handles the 36 Tactical Triads, calculates Advantage/Disadvantage, rolls the virtual dice, and strictly enforces the  "Scars & Stars"  progression system by telling the GM App when to award a Pip. Tech Stack:  Python 3.11+, FastAPI, Pydantic. 2. Python def   check_for_pips (raw_die:  int , is_success:  bool , is_life_or_death:  bool ) -> str:      if  raw_die ==  20 :          return   "STAR"   # Perfection under fire      if  raw_die ==  1 :          return   "SCAR"   # Disastrous failure, painful lesson learned      if  is_success  and  is_life_or_death:          return   "SURVIVOR"   # MVP moment           return   None   # Routine checks yield no progression B. BLUEPRINT 07: The Rules Manager (The Clash Engine) 1. System Overview Definition:  A stateless Python microservice. This is the crown jewel of your tactical combat. It completely replaces the boring "Armor Class" system of D&D. It takes an Attacker and a Defender, rolls for both, compares the math, and outputs the exact state change (Crushing Win, Scrape, Reversal, or Deadlock). Tech Stack:  Python 3.11+, FastAPI, Pydantic. 2. Internal Logic (How to Code It) A. The Clash Calculator (core/clash_calculator.py) How to code it: This is the pure mathematical translation of your Contested Combat design doc. Python def   resolve_clash (req: ClashRequest) -> ClashResolution:      # 1. Roll the dice for both sides and add their burned Stamina/Focus     atk_total = roll_d20() + req.attacker.attack_or_defense_pool + req.attacker.stamina_burned     def_total = roll_d20() + req.defender.attack_or_defense_pool + req.defender.stamina_burned          margin = atk_total - def_total     res = ClashResolution(         stamina_deducted_attacker=req.attacker.stamina_burned,         stamina_deducted_defender=req.defender.stamina_burned     )           # 2. Determine Outcome based on the Margin      if  margin >=  5 :         res.clash_result =  "CRUSHING_WIN"          # Roll full weapon damage + trigger chaos/tactical effect         res.defender_hp_change = -roll_dice(req.attacker.weapon_damage_dice)               elif   1  <= margin <=  4 :         res.clash_result =  "SCRAPE"          # Half damage, no tactical advantage         res.defender_hp_change = -(roll_dice(req.attacker.weapon_damage_dice) //  2 )               elif  margin <  0 :         res.clash_result =  "REVERSAL"          # Defender wins the clash! They get a free counter-attack or chip damage.         res.attacker_hp_change = - 2   # Chip damage               else :  # margin == 0         res.clash_result =  "DEADLOCK"          # Both parties are locked. If chaos is high, trigger a world event.          if  req.chaos_level >  3 :             res.chaos_effect_triggered =  "Reality Warp: Ground shifts, both fall Prone."                   return  res B. The Injury Applier (core/injury_applier.py) How to code it: If a character drops to 0 HP during the Clash, this script intercepts the result and translates it into your Dual-Track system. Logic:  If defender_hp_change reduces HP to 0 -> set defender_injury_applied = "1 Major Body Injury (Shattered Bone)". 5.
